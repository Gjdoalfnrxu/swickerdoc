The Problem

Standard models don't enforce the accounting

In a conventional macro model, you can build a scenario where government spending rises, private savings rise, and the current account improves simultaneously — with no sector in the model footing the bill. The accounting is violated. The model doesn't know.

Stock-Flow Consistent (SFC) modelling, developed by Wynne Godley and Marc Lavoie, imposes the full national accounts constraint at the model level. The balance sheet matrix and transaction flow matrix must close before a simulation runs. If they don't, you find out at compile time.

The approach is used by central banks, the Levy Economics Institute, and academic researchers in the post-Keynesian tradition for precisely this reason: it rules out accounting errors masquerading as economic results.

Balance Sheet Closure

Every financial asset maps to a matching liability across sectors. Validated at compile time.

Stock-Flow Consistency

Flows accumulate into stocks each period. Deposits, loans, bonds evolve coherently.

Endogenous Money

Banks extend loans and create deposits as a result. Money supply emerges from behaviour.

Sectoral Balances Identity

Government deficit + private surplus + current account = 0. Enforced, not assumed.

Technical

How it works

1

Define sectors, variables, and equations

Specify institutional sectors, stock and flow variables, behavioural equations (e.g. C = alpha1 * YD + alpha2 * V[-1]), parameters, and the balance sheet matrix mapping assets to holder/issuer sector pairs.

2

Compile and validate

The PEG grammar parser tokenises equations, extracts the dependency graph, runs topological sort, and compiles to bytecode. Balance sheet closure is validated. Compilation errors are reported before any simulation is attempted.

3

Simulate with shocks

Gauss-Seidel iteration solves the system forward period by period until convergence. Apply parameter or variable shocks at any period — level changes, delta changes, or percent changes. Real-time progress via WebSocket.

4

Analyse and compare

Interactive results across time series, balance sheet evolution, deviation, and multi-scenario overlay tabs. Save scenarios, run comparisons, export results. The same engine runs in the browser via WebAssembly for instant local simulation.
