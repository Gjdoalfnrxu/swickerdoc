setState through React Context

How the tsq bridge tracks setters across createContext / Provider / useContext

PR #165 (round-2) + PR #167 (round-3) — bridge/tsq_react.qll

What we want to catch


const ViewerStateActions = createContext(null);

function ViewerProvider({ children }) {
  const [zoom, setZoom] = useState(1);
  return (
    <ViewerStateActions.Provider value={{ setZoom }}>
      {children}
    </ViewerStateActions.Provider>
  );
}

function useViewerActions() { return useContext(ViewerStateActions); }

function ZoomButton() {
  const { setZoom } = useViewerActions();
  setZoom(prev => { setOther(p => ...); return prev + 1; });  // match
}

The inner setZoom call needs to be recognised as a useState setter even though it arrived via Context.

The pipeline

useState                       Provider                    Consumer
────────                       ────────                    ────────
useStateSetterSym(s) ──┐
                       ▼
              contextProviderField(ctx, "setX", s)
                       │            ▲
                       │            │  resolveToObjectExpr
                       │            │  objectLiteralFieldThroughSpread
                       ▼            │
              contextSymLink(provCtx, consCtx)
                       │
                       ▼
              contextDestructureBinding(consCtx, "setX", paramSym)
                       │
                       ▼
              contextSetterAliasStep(s, paramSym) ──► alias closure

Every box is a separate predicate. Every predicate is sized independently by the planner.

Step 1 — recognise the context handle


predicate contextSym(int sym) {
  exists(int initExpr, int createCtxSym |
    VarDecl(_, sym, initExpr, _) and
    CallCalleeSym(initExpr, createCtxSym) and
    ImportBinding(createCtxSym, "react", "createContext")
  )
}
  • Name-based on the import binding.
  • Aliased imports (createContext as cc) work — callee sym still resolves to the imported binding.
  • Namespace shape (React.createContext(...)) deferred — same gap as useState's namespace shape in round-1.

Step 2 — find the value object on a Provider


predicate contextProviderValueObject(int ctxSym, int objExpr) {
  exists(int elem, int tagNode, int valueAttrExpr |
    JsxElement(elem, tagNode, _) and
    FieldRead(tagNode, ctxSym, "Provider") and        // <Ctx.Provider ...>
    JsxAttribute(elem, "value", valueAttrExpr) and
    jsxAttrValueObject(valueAttrExpr, objExpr) and
    ObjectLiteralField(objExpr, _, _)
  )
}

Provider tag is a MemberExpression — pivot on the FieldRead with fieldName = "Provider". jsxAttrValueObject tolerates either the bare ObjectLiteral or the JsxExpression wrapper.

Step 2.5 — round-3: the value isn't always inline


const actions = { setZoom };
<Ctx.Provider value={actions} />                     // variable indirection
<Ctx.Provider value={{ ...base, setPan }} />         // spread
<Ctx.Provider value={{ ["set" + "Zoom"]: setZoom }} />  // computed key

resolveToObjectExpr(valueExpr, objExpr) — four named branches, unioned by or-of-calls (workaround for #166):

BranchShape
DirectvalueExpr = objExpr and objExpr is an ObjectLiteral
WrappedContains(valueExpr, objExpr) (JsxExpression wrapper)
VarD1value={X}, const X = { ... }
VarD2value={X}, const X = Y; const Y = { ... }

Hand-unrolled to depth 2. Circular const a = b; const b = a; chains terminate by depth, not by cycle reasoning.

Step 2.75 — spread fields


predicate objectLiteralFieldThroughSpread(int objExpr, string fieldName, int valueExpr) {
  objectLiteralFieldOwn(objExpr, fieldName, valueExpr)
  or
  objectLiteralFieldSpreadD1(objExpr, fieldName, valueExpr)
  or
  objectLiteralFieldSpreadD2(objExpr, fieldName, valueExpr)
}
  • D1: { ...base, x } — base is direct or VarDecl-bound.
  • D2: { ...{ ...deeper } } — direct/var × direct/var = 4 named helpers.

Why split into named helpers: inline (A or B) bodies desugar to _disj_N predicates the planner can't size — same pathology as round-1's functionContainsStar.

Step 3 — link Provider symbol to Consumer symbol


predicate contextSymLinkSame(int prov, int cons) {
  contextSym(prov) and cons = prov              // single-file
}
predicate contextSymLinkCrossFile(int prov, int cons) {
  exists(string name |
    ExportBinding(name, prov, _) and
    ImportBinding(cons, _, name)                // cross-file by export name
  )
}
predicate contextSymLink(int prov, int cons) {
  contextSymLinkSame(prov, cons) or contextSymLinkCrossFile(prov, cons)
}

Round-3 fix: round-2's same-file branch had an unbound equality inside the disjunction body. Its only fixture happened to use the cross-file path so the planner-can't-ground bug never surfaced. Now anchored on contextSym(prov).

Step 4 — recognise the consumer side


predicate useContextCall(int call, int ctxSym) { /* useContext(Ctx) */ }

predicate hookIndirectionD1(int hookFn, int ctxSym) {
  exists(int retExpr, int innerCall |
    ReturnStmt(hookFn, _, retExpr) and
    ExprIsCall(retExpr, innerCall) and
    useContextCall(innerCall, ctxSym)
  )
}
predicate hookIndirectionD2(int hookFn, int ctxSym) { /* hook → hook → useContext */ }

predicate useContextCallSiteResolvesContext(int call, int ctxSym) {
  useContextCall(call, ctxSym)
  or  /* call invokes a hook function whose body returns useContext(ctx) */
  or  /* same, but the hook is imported from another module */
}

Hand-unrolled to depth 2 — same planner-sizing rationale as everywhere else.

Step 5 — destructure the consumer result


predicate contextDestructureBinding(int ctxSym, string fieldName, int paramSym) {
  exists(int varDecl, int parent, int initExpr, int callExpr, int call |
    VarDecl(varDecl, _, initExpr, _) and
    Contains(varDecl, parent) and
    DestructureField(parent, fieldName, _, paramSym, _) and
    (callExpr = initExpr or Cast(initExpr, callExpr)) and  // tolerate useCtx()!
    ExprIsCall(callExpr, call) and
    useContextCallSiteResolvesContext(call, ctxSym)
  )
}

paramSym is the Identifier on the LHS of const { setZoom } = .... The field-name match between this and contextProviderField disambiguates which field the destructure is reading.

Step 6 — the alias step


predicate contextSetterAliasStep(int valueSym, int paramSym) {
  contextSetterAliasStepR2(valueSym, paramSym)              // round-2 path
  or
  contextSetterAliasStepR3DirectSpread(valueSym, paramSym)  // round-3 inline+spread
  or
  contextSetterAliasStepR3VarIndirect(valueSym, paramSym)   // round-3 variable
}

Each branch shares this skeleton:


exists(int provCtx, int consCtx, string fieldName |
  contextSym(provCtx) and
  contextProviderFieldR*(provCtx, fieldName, valueSym) and
  contextSymLink(provCtx, consCtx) and
  contextDestructureBinding(consCtx, fieldName, paramSym)
)

Three named branches, top-level or-of-calls. Same #166 workaround.

Step 7 — fold into the alias closure


predicate setterAliasStepAny(int valueSym, int paramSym) {
  setterAliasStep(valueSym, paramSym)         // round-1 JSX prop hop
  or
  contextSetterAliasStep(valueSym, paramSym)  // round-2/3 context hop
}

predicate useStateSetterAliasV2(int sym) {
  useStateSetterSym(sym)
  or  exists(int s0 | useStateSetterSym(s0) and setterAliasStepAny(s0, sym))
  or  exists(int s0, int s1 | ... two hops ...)
  or  exists(int s0, int s1, int s2 | ... three hops ...)
}

Hand-unrolled to depth 3. Mixed chains (prop → context, or context → prop) fall out for free because both step kinds live in setterAliasStepAny.

Step 8 — diagnostic filter


predicate isContextAliasedSetterSym(int sym) { /* at least one ctx hop */ }
predicate useStateSetterContextAliasCall(int call) { /* same, at call site */ }

The query needs to surface only matches that are diagnostically about the context channel — a pure direct-form match would already fire on the round-1 query.

isContextAliasedSetterSym requires ≥1 contextSetterAliasStep in the chain rooted at a useStateSetterSym. Hand-unrolled depth 3.

Step 9 — the query predicate


predicate setStateUpdaterCallsOtherSetStateThroughContext(int call, int line) {
  setStateUpdaterCallsOtherSetStateThroughContext_outerCtx(call, line)
  or
  setStateUpdaterCallsOtherSetStateThroughContext_innerCtx(call, line)
}
  • outerCtx: outer setter is context-aliased, inner can be anything.
  • innerCtx: inner setter is context-aliased, outer can be anything.

Both share the setStateUpdaterCallsOtherSetState skeleton: outer call has a function-literal first arg, functionContainsStar reaches innerCall, outerSym != innerSym.

Why so many named helpers?

1. Disjunction poisoning (#166). An inline (A or B) body desugars to a synthesised _disj_N predicate. The planner can't size _disj_N predicates → defaults to 1000 → picks a Cartesian-heavy join order → blows the 5M binding cap on Mastodon-scale corpora.

Fix: hoist each branch to a named predicate, union at the top with or-of-calls. Each named predicate is sized independently by the trivial-IDB pre-pass + sampling estimator (P2b).

2. Recursive-IDB sizing. Recursive predicates aren't materialised by the trivial-IDB pre-pass — same default-1000 hint problem.

Fix: hand-unroll to a fixed depth (3 for closures, 2 for spread). Real prop-drilling and spread-chain depths in production React are shallow.

Soundness/precision deferrals

Documented in code, not silently elided:

  • React.createContext(...) namespace-import shape — out of v1.
  • Multi-Provider disambiguation is over-approximate — two unrelated Providers for the same context produce false positives.
  • Hook-indirection chains deeper than 2 — deferred.
  • let-reassignment last-write semantics — over-approximated as union of all reachable RHS object literals.
  • Helper-call Provider values (value={makeActions()}) — out of scope in v3. (Round-4 in flight to cover the hook-return shape.)
  • Spread chains deeper than 2 — out of scope.

Recap

  1. contextSym — the handle (createContext result).
  2. contextProviderField* — what the Provider exposes (own / spread / var-indirect).
  3. contextSymLink — Provider sym ↔ Consumer sym.
  4. useContextCallSiteResolvesContext — the consumer call.
  5. contextDestructureBinding — the LHS Identifier on the consumer side.
  6. contextSetterAliasStep — one context hop.
  7. setterAliasStepAny + useStateSetterAliasV2 — fold into the closure.
  8. useStateSetterContextAliasCall — diagnostic "≥1 ctx hop" filter.
  9. setStateUpdaterCallsOtherSetStateThroughContext — outer or inner call is context-aliased.

Every layer is named. Every or is or-of-calls. Every closure is hand-unrolled. The planner can size everything.
